PARTE B — ANTI-FRAUD

6. lib/antifraud.ts — Layer anti-fraud:
   
   a) Device fingerprinting (lato client):
      - Installa: npm install @fingerprintjs/fingerprintjs
      - Crea components/Fingerprint.tsx: componente client che al mount genera fingerprint e lo salva in cookie "dfp" (30 giorni)
      - Includi questo componente nel root layout
   
   b) Validazioni server-side (tutte in lib/antifraud.ts):
      - checkDuplicateDevice(fingerprint: string): se esiste altro User con stesso fingerprint → blocca registrazione
      - checkReferralRateLimit(userId: string): max 10 referral qualificati per giorno per user. Se superato → non qualificare, logga warning
      - checkIpClustering(ip: string): se > 5 registrazioni dallo stesso /24 IP nelle ultime 24h → flag user come suspicious
      - checkWithdrawalEligibility(userId: string): 
        * Balance >= $5
        * Account creato da almeno 48h
        * User non flaggato come suspicious
        * Max 1 withdrawal per 24h
   
   c) Aggiungi campo "suspicious" Boolean @default(false) al model User in Prisma
   d) Aggiungi campo "deviceFingerprint" String? al model User
   
   e) Integra i check:
      - In verify-otp: chiama checkDuplicateDevice prima di creare user
      - In referral qualification: chiama checkReferralRateLimit
      - Crea /api/wallet/withdraw endpoint che chiama checkWithdrawalEligibility

7. app/api/wallet/withdraw/route.ts — Endpoint prelievo:
   - POST, auth required, body: { amount: number, address: string }
   - Chiama checkWithdrawalEligibility
   - Se ok: decrementa balance, crea WalletTransaction type "withdrawal", status "pending"
   - Per ora: NON esegui transazione blockchain reale. Solo update DB.
   - Ritorna { success: true, txId: transaction.id }
